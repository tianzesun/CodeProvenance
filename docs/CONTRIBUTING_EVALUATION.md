# Formal Evaluation Specification

This document defines the mandatory standards for performing and reporting evaluations within the CodeProvenance project. Adherence to these standards is required for all research and publication-grade results.

## 1. Reproducibility Requirements

All evaluation results must be accompanied by a `ReproducibilityReport` generated by the `src/evaluation/reproducibility.py` module. This report must include:

- **System Context**: OS, Python version, hardware specifications.
- **Engine Configuration**: All weights, thresholds, and hyper-parameters used.
- **Dataset Versioning**: Content hashes and registration identifiers.
- **Result Hashes**: Cryptographic signatures of the raw metrics.

## 2. Dataset Governance

Datasets must be registered via the `DatasetGovernance` layer (`src/evaluation/dataset/governance.py`).

- **Fingerprinting**: Every file must be hashed.
- **Contamination Check**: No overlaps are permitted between training/fine-tuning sets and test benchmarks.
- **Deduplication**: Identical code fragments across datasets must be explicitly flagged and handled.

## 3. Statistical Rigor

Mean scores (P, R, F1) are insufficient for publication-grade credibility.

- **Confidence Intervals**: All metrics must be reported with 95% bootstrap confidence intervals.
- **Significance Testing**: System comparisons must use Paired Bootstrap Tests or McNemar’s Test (for binary outcomes).
- **Arbitration**: Multi-engine fusion must use the `BayesianArbitrator` to account for uncertainty and engine disagreement.

## 4. Metric Reporting

Mandatory metrics for every benchmark:

| Metric | Purpose |
|--------|---------|
| **F1-Score** | Primary harmonic mean of Precision and Recall. |
| **AUC-ROC** | Discriminative power across all thresholds. |
| **ECE** | Calibration error (reliability of scores). |
| **Agreement Index** | Measure of engine consensus. |

## 5. Scope Freeze

The following areas are currently under **scope freeze**. NO additions are permitted without explicit architectural review:

- **Visualization Styles**: Do not add new chart types or dashboard views.
- **Detection Engines**: The core 6 engines are fixed.
- **Smart Interpretation**: Avoid NLP-generated "explanations". Stick to "Forensic Evidence" (raw scores, arbitration metrics).

## 6. CI Gates

CI pipelines will enforce the following gates:
- **No Performance Regression**: F1 must not drop >1% from baseline.
- **Integrity Check**: Benchmark datasets must match registered hashes.
- **Documentation**: New code must include test cases for evaluation logic.
